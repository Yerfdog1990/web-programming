# Fonts in CSS

Fonts are a fundamental part of web design. They affect:

* Readability
* User experience
* Professional appearance
* Visual hierarchy

In CSS, fonts are controlled primarily using the `font-family` property.

---

# 2.1 The `font-family` Property

The `font-family` property specifies which font should be used to display text.

Because not all users have the same fonts installed, you can define **multiple fonts separated by commas**. The browser will try them in order and use the first one available.

This is called a **font stack**.

### Example: Multiple Fonts

```css
font-family: "Helvetica Neue", Helvetica, Arial, sans-serif;
```

### How It Works

1. Try `"Helvetica Neue"`
2. If not available → Try `Helvetica`
3. If not available → Try `Arial`
4. If none are available → Use any available `sans-serif` font

---

### Example Usage

```html
<!DOCTYPE html>
<html>
<head>
  <style>
    body {
      font-family: "Helvetica Neue", Helvetica, Arial, sans-serif;
    }
  </style>
</head>
<body>
  <p>This text is displayed using Helvetica Neue, Helvetica, or Arial fonts.</p>
</body>
</html>
```

---

# 2.2 Serif Font Family

**Serif fonts** have small decorative lines (called serifs) at the ends of letters.

### Characteristics:

* Traditional
* Formal
* Often used in books and print media

### Examples of Serif Fonts:

* Times New Roman
* Georgia
* Garamond
* Baskerville

---

### Example Usage

```html
<!DOCTYPE html>
<html>
<head>
  <style>
    body {
      font-family: "Times New Roman", serif;
    }
  </style>
</head>
<body>
  <h1>This is a heading</h1>
  <p>This is a paragraph using a serif font.</p>
</body>
</html>
```

If `"Times New Roman"` is not available, the browser will use the default `serif` font.

---

# 2.3 Sans-serif Font Family

**Sans-serif fonts** do NOT have decorative strokes at the ends of letters.

### Characteristics:

* Clean
* Modern
* Minimalist
* Highly readable on screens

### Examples of Sans-serif Fonts:

* Arial
* Helvetica
* Verdana
* Open Sans

---

### Example Usage

```html
<!DOCTYPE html>
<html>
<head>
  <style>
    body {
      font-family: "Arial", sans-serif;
    }
  </style>
</head>
<body>
  <h1>This is a heading</h1>
  <p>This is a paragraph using a sans-serif font.</p>
</body>
</html>
```

Sans-serif fonts are widely used in web design due to excellent screen readability.

---

# 2.4 Monospace Font Family

**Monospace fonts** give every character the same width.

For example:

* The letter "i" takes the same space as "W"

### Characteristics:

* Equal character width
* Perfect alignment
* Ideal for code and tables

### Examples of Monospace Fonts:

* Courier New
* Consolas
* Monaco
* Lucida Console

---

### Example Usage

```html
<!DOCTYPE html>
<html>
<head>
  <style>
    body {
      font-family: "Courier New", monospace;
    }
  </style>
</head>
<body>
  <h1>This is a heading</h1>
  <p>This is a paragraph using a monospace font.</p>
  <pre>
function helloWorld() {
  console.log("Hello, world!");
}
  </pre>
</body>
</html>
```

Monospace fonts are commonly used for:

* Code snippets
* Technical documentation
* Terminal output

---

# Combining Font Families in Web Design

Good web design often uses:

* **Serif fonts** for headings
* **Sans-serif fonts** for body text
* **Monospace fonts** for code

This creates contrast and improves readability.

---

# Exercise 1: Font Family

## Requirements

* `h1` should use:

    * `"Georgia"`
    * `"Times New Roman"`
    * `serif`

* `p` should use:

    * `"Verdana"`
    * `"Arial"`
    * `sans-serif`

* Fonts must be listed in **descending order of priority**.

---

## ✅ Sample Solution

```html
<!DOCTYPE html>
<html>
<head>
  <style>
    h1 {
      font-family: "Georgia", "Times New Roman", serif;
    }

    p {
      font-family: "Verdana", "Arial", sans-serif;
    }
  </style>
</head>
<body>

  <h1>This is a heading</h1>
  <p>This is a paragraph styled with a sans-serif font stack.</p>

</body>
</html>
```

# Exercise 2: Monospace Fonts

## Requirements

* `h4` should use `"Courier New"`
* `p` should use `"Courier New"`
* `pre` should use `"Courier New"`
* All elements inside `.monospace-container` must use `"Courier New"`

---

## ✅ Sample Solution

```html
<!DOCTYPE html>
<html>
<head>
  <style>
    .monospace-container {
      font-family: "Courier New", monospace;
    }
  </style>
</head>
<body>

  <div class="monospace-container">
    <h4>This is a heading</h4>
    <p>This is a paragraph inside the monospace container.</p>
    <pre>
function example() {
  console.log("Monospace font applied!");
}
    </pre>
  </div>

</body>
</html>
```

### Why This Works

Instead of styling each element separately (`h4`, `p`, `pre`), we apply the font to the parent container:

```css
.monospace-container {
  font-family: "Courier New", monospace;
}
```

All child elements inherit the font automatically.

---

# Key Takeaways

* ✔ `font-family` defines which font is used
* ✔ Always provide fallback fonts
* ✔ Use commas to separate fonts
* ✔ Generic families: `serif`, `sans-serif`, `monospace`
* ✔ Use font stacks for compatibility
* ✔ Combine font families for better design

Mastering font families is essential for creating clean, readable, and professional websites.
